## 新店铺初始化 SOP（用户触发后 Claude 执行）

### Step 1 · 问 4 个问题

按顺序问，**用户答完才进 Step 2**：

1. 店铺品牌全名？（中文，如"杭州 XX 包装有限公司"）
2. 1688 二级域名？（如 `xxbao.1688.com` 或 `shop1234567890.1688.com`）
3. 主营类目结构？（按比例，如"塑料 60% / 纸盒 30% / 综合 10%"）
4. 商业化对待？（"会" / "可能" / "纯自用" / "不知道"）

> 第 4 题决定走 **完整路径** vs **轻量路径**（参考主 CLAUDE.md "新项目检查清单"）。

### Step 2 · 项目骨架建立

```bash
# 英文项目名规则：店铺主体英文/拼音 + "-1688体检"
# 例："杭州 XX 包装" → "hzxx-1688体检"
mkdir -p ~/projects/<英文名>/{01-诊断报告,02-数据快照,03-执行SOP,04-周度复盘,05-对接记录,06-人群投放配置,07-商品管理,08-参考资料,docs,scripts}
cd ~/projects/<英文名> && git init -b main
```

### Step 3 · 套模板（cp + 占位符替换）

```bash
# 复制单店铺 CLAUDE.md 模板
cp ~/projects/_1688店铺体检引擎/模板/单店铺_CLAUDE.md.template ./CLAUDE.md

# 复制报告模板
cp ~/projects/_1688店铺体检引擎/模板/报告模板/月度体检报告_V1模板.html ./01-诊断报告/

# 复制 7 份数据快照模板
cp ~/projects/_1688店铺体检引擎/模板/数据快照模板/*.md ./02-数据快照/

# 复制行动表模板
cp ~/projects/_1688店铺体检引擎/模板/行动表/月度行动表模板.md ./03-执行SOP/

# 占位符替换（用 sed -i '' 在 macOS）
find . -type f \( -name "*.md" -o -name "*.html" \) -exec sed -i '' \
  -e "s/{{店铺名}}/$店铺名/g" \
  -e "s/{{二级域名}}/$二级域名/g" \
  -e "s/{{主营类目}}/$主营类目/g" \
  -e "s/{{创建日期}}/$(date +%Y-%m-%d)/g" \
  -e "s/{{年框}}/未签 SKA/g" {} \;
```

### Step 4 · 注册 ccd 缩写

在主 CLAUDE.md（`~/.claude/projects/-Users-jindaojian/CLAUDE.md`）"已注册缩写"列表里加：

```
- `ccd <店铺缩写>` → ~/projects/<英文名>/（XX 1688 店铺月度体检）
```

### Step 5 · 写 memory 索引

在主 memory（`~/.claude/projects/-Users-jindaojian/memory/`）：

```bash
echo "- [<店铺名>](project_<英文名>.md) — 1688 店铺月度体检，~/projects/<英文名>/" >> MEMORY.md
```

并写 memory 文件 `project_<英文名>.md`。

### Step 6 · 首次 commit

```bash
git add .
git commit -m "feat: <店铺名> 1688 月度体检项目初始化（套引擎模板 V1）"
```

### Step 7 · 报告 owner

```
✅ 项目骨架就位
- 路径：~/projects/<英文名>/
- 进入：ccd <缩写>
- 触发首次月度体检：在项目内说"跑月度体检"

```

## 模板更新流程（引擎升级）

当用户说"1688 引擎升级 / 模板优化"：

1. 在 `~/projects/_1688店铺体检引擎/` 内修改对应文件
2. 已落地的店铺**不自动同步**（避免破坏个性化）
3. 通过版本号追溯：每次升级 commit 标 `engine-v1.x → engine-v1.y`
4. 老店铺手动 cherry-pick 升级：用户决定要不要拉取
