# 1688 店铺月度体检引擎

> 输入店铺名 + 1688 二级域名 → 自动产出完整项目骨架 + 数据采集 + 报告生成
>
> **零手工**：任何 1688 店铺丢进来即用。

## 核心定位

- **引擎层**（本项目）：方法论 + 模板 + 触发词，**通用且与具体客户解耦**
- **交付层**（每个店铺独立项目）：客户数据 + 月度报告 + 行动表

第一个落地客户：金驰包装（`~/projects/jc-1688/`）
后续客户：直接 `/新建 1688 店铺体检 <名称>` 触发引擎

## 三步上手

### Step 1 · 在任意 Claude 会话说触发词

```
新建 1688 店铺体检 杭州 XX 包装
```

或

```
帮我接入新店铺到 1688 体检引擎
```

### Step 2 · Claude 问 4 个问题（你答）

1. 店铺品牌名（中文，如"杭州 XX 包装有限公司"）
2. 1688 二级域名（如 `xxbao.1688.com` 或 `shop123456.1688.com`）
3. 主营类目结构（如"塑料 60% / 纸盒 30% / 综合 10%"）
4. 是否商业化对待？（影响走完整 / 轻量 / 保守路径）

### Step 3 · Claude 自动完成

- 建 `~/projects/<店铺英文名>-1688体检/` 项目目录
- 套模板：CLAUDE.md / 报告 HTML / 7 份数据快照 / 行动表
- git init + 注册 ccd 缩写
- 等你说"跑月度体检"触发抓取

## 商业化沉淀

- 引擎层与交付层完全解耦
- 未来可作"1688 SKA 大客户诊断 SaaS"
- 第一个落地：金驰包装（验证完成）
- 第 N 个落地：复制粘贴成本 ≈ 0

## 红线（不可跨）

- 禁止自动过 1688 验证码（撞码立即让用户手动）
- 禁止自动定时跑（用户召唤式触发）
- 禁止自动点"投放"按钮（碰钱红线）
- 禁止 git push --force / --no-verify

## 触发词速查

| 触发词 | 动作 |
|--------|------|
| 新建 1688 店铺体检 \<名称\> | 套模板初始化新店铺项目 |
| 跑 \<店铺\> 1688 月度体检 | 进入项目跑月度体检（数据抓取 + 报告生成）|
| 跑 1688 周报 | 在当前店铺项目里跑周度复盘 |
| 1688 引擎升级 | 优化模板 / 思维导图 / SOP（影响所有未来店铺）|
