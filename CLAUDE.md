## ⛔ V2 核心红线（不可跨）

### 红线 1 · 禁止跨客户对比

- 每家店铺独立体检
- **绝对禁止**拿其他客户店铺当本店参照（金驰 vs 艺漫坊不行 / 艺漫坊 vs 任意新店也不行）
- 每份月度报告里不允许出现"vs <其他客户店铺名>"这样的句式
- 跨客户对比唯一允许场景：**引擎层方法论文档**（说明 V1/V2 演进时可引用案例）

### 红线 2 · 同行对比必须三梯度

- 每份月度体检报告**必须**抓同行三梯度数据作为对比组
- 梯度 A · **普通商家**（同等级 / 同类目 / 同钻级）抓 3-5 家
- 梯度 B · **实力商家**（带"实力商家"标签 · 学习对标）抓 2-3 家
- 梯度 C · **超级工厂 / 金牌制造**（行业头部 · 仰望标杆）抓 1-2 家
- 共 6-10 家同行，覆盖 3 个梯度

### 红线 3 · 同行对比的 4 维度

- **优惠力度**：满减 / 折扣 / 优惠券 / 包邮门槛
- **营销手段**：活动报名 / 直播 / 引流款 / VIP 会员体系
- **价格策略**：价格区间 / 阶梯价 / 起订量分级
- **服务承诺**：发货时效 / 包赔 / 退换货政策

### 红线 4 · 报告模块 6 必须呈现三梯度对比矩阵

- 模块 6 标题改为"竞品 & 同行三梯度对比"
- 必须有对比表：本店 vs 普通梯度 vs 实力梯度 vs 超级工厂
- 必须给出三梯度结论：差距 / 学什么 / 长期目标

### ⛔ 红线 5 · IP 授权风险（V3 必加 · 写进每份报告）

- **绝对禁止**建议本店"立即上 IP 词 SKU"（鬼灭之刃 / 时光代理人 / TNT 少年团 / 谷子二次元 / 名侦探柯南 / 原神 / 哈利波特 / 迪士尼 / Sanrio 三丽鸥 / 漫威 / 火影 / 海贼王 / 排球少年 / 咒术回战 / 蓝色监狱 等任何已授权 IP）
- 同行用 IP 词的高销量背后是**授权或灰色操作**，正规商家不能复制
- 替代建议：**合规通用风口词**
  - 节日场景：中秋 / 春节 / 国庆 / 母亲节 / 毕业季
  - 校园应援：校徽 / 班服 / 社团 / 毕业纪念
  - 婚庆礼盒：伴手礼 / 喜糖 / 婚礼布置
  - 企业定制：logo / 公司礼品 / 年会纪念
  - 通用风格：Q 版 / 卡通 / 二次元周边（无 IP 名称）/ 流沙 / UV 印刷
  - 玩法品类：盲盒 / 立牌 / 钥匙扣 / 冰箱贴 / 徽章
- 报告里发现同行用 IP 词时，必须标注"侵权风险 ⚠️"，不能作为本店行动建议

## 新店铺初始化 SOP（用户触发后 Claude 执行）

### Step 1 · 问 4 个问题

按顺序问，**用户答完才进 Step 2**：

1. 店铺品牌全名？（中文，如"杭州 XX 包装有限公司"）
2. 1688 二级域名？（如 `xxbao.1688.com` 或 `shop1234567890.1688.com`）
3. 主营类目结构？（按比例，如"塑料 60% / 纸盒 30% / 综合 10%"）
4. 商业化对待？（"会" / "可能" / "纯自用" / "不知道"）

> 第 4 题决定走 **完整路径** vs **轻量路径**（参考主 CLAUDE.md "新项目检查清单"）。

### Step 2 · 项目骨架建立

```bash
# 英文项目名规则：店铺主体英文/拼音 + "-1688体检"
# 例："杭州 XX 包装" → "hzxx-1688体检"
mkdir -p ~/projects/<英文名>/{01-诊断报告,02-数据快照,03-执行SOP,04-周度复盘,05-对接记录,06-人群投放配置,07-商品管理,08-参考资料,docs,scripts}
cd ~/projects/<英文名> && git init -b main
```

### Step 3 · 套模板（cp + 占位符替换）

```bash
# 复制单店铺 CLAUDE.md 模板
cp ~/projects/_1688店铺体检引擎/模板/单店铺_CLAUDE.md.template ./CLAUDE.md

# 复制报告模板
cp ~/projects/_1688店铺体检引擎/模板/报告模板/月度体检报告_V1模板.html ./01-诊断报告/

# 复制 7 份数据快照模板
cp ~/projects/_1688店铺体检引擎/模板/数据快照模板/*.md ./02-数据快照/

# 复制行动表模板
cp ~/projects/_1688店铺体检引擎/模板/行动表/月度行动表模板.md ./03-执行SOP/

# 占位符替换（用 sed -i '' 在 macOS）
find . -type f \( -name "*.md" -o -name "*.html" \) -exec sed -i '' \
  -e "s/{{店铺名}}/$店铺名/g" \
  -e "s/{{二级域名}}/$二级域名/g" \
  -e "s/{{主营类目}}/$主营类目/g" \
  -e "s/{{创建日期}}/$(date +%Y-%m-%d)/g" \
  -e "s/{{年框}}/未签 SKA/g" {} \;
```

### Step 4 · 注册 ccd 缩写

在主 CLAUDE.md（`~/.claude/projects/-Users-jindaojian/CLAUDE.md`）"已注册缩写"列表里加：

```
- `ccd <店铺缩写>` → ~/projects/<英文名>/（XX 1688 店铺月度体检）
```

### Step 5 · 写 memory 索引

在主 memory（`~/.claude/projects/-Users-jindaojian/memory/`）：

```bash
echo "- [<店铺名>](project_<英文名>.md) — 1688 店铺月度体检，~/projects/<英文名>/" >> MEMORY.md
```

并写 memory 文件 `project_<英文名>.md`。

### Step 6 · 首次 commit

```bash
git add .
git commit -m "feat: <店铺名> 1688 月度体检项目初始化（套引擎模板 V1）"
```

### Step 7 · 报告 owner

```
✅ 项目骨架就位
- 路径：~/projects/<英文名>/
- 进入：ccd <缩写>
- 触发首次月度体检：在项目内说"跑月度体检"
- 抓取 SOP：参考引擎方法论思维导图
```

## 模板更新流程（引擎升级）

当用户说"1688 引擎升级 / 模板优化"：
