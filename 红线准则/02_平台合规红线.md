# 平台合规红线（1688 平台规则 + 法律层面）

> 4 条合规红线，全部围绕「不让店铺背风险」。

## R-005 · IP 授权风险（艺漫坊事件触发 · V3 写死）

**一句话**：禁止建议本店上 IP 词 SKU；同行 IP 词必须标 ⚠️侵权风险。

**详细解释（owner 2026-04-26 强调）**：
- 同行用 IP 词销量高 = **背后是授权或灰色操作**
- 正规商家无授权直接上 IP 词 = 平台投诉 / 罚款 / 下架 / 永封
- 艺漫坊 V1 报告曾建议"上鬼灭之刃 / 时光代理人"→ owner 砍掉，V3 引擎写死禁止

**禁用 IP 关键词清单**（持续维护）：
| 类型 | IP |
|---|---|
| 二次元 | 鬼灭之刃 / 时光代理人 / 名侦探柯南 / 原神 / 排球少年 / 咒术回战 / 蓝色监狱 / 火影 / 海贼王 |
| 偶像 | TNT 少年团 / 时代少年团 / 任何明星名 |
| IP 集合词 | 谷子二次元 / 谷子周边 / 二次元正版 |
| 国际 IP | 哈利波特 / 迪士尼 / Sanrio 三丽鸥 / 漫威 / DC |

**出错示例**：
- ❌ "P0 立即上鬼灭之刃 SKU"
- ❌ "建议铺时光代理人钥匙扣"
- ❌ "学同行的 TNT 少年团徽章"（不分析为什么不能学 = 不及格）

**替代方案 · 合规通用风口词**：

| 类别 | 风口词 |
|---|---|
| **节日场景** | 中秋 / 春节 / 国庆 / 母亲节 / 父亲节 / 毕业季 / 教师节 |
| **校园应援** | 校徽 / 班服 / 社团 / 毕业纪念 / 同学录 |
| **婚庆礼盒** | 伴手礼 / 喜糖 / 婚礼布置 / 婚礼小卡 |
| **企业定制** | logo / 公司礼品 / 年会纪念 / VI 周边 |
| **通用风格** | Q 版 / 卡通 / 二次元周边（无具体 IP 名）/ 流沙 / UV 印刷 / 钻石切割 |
| **玩法品类** | 盲盒 / 立牌 / 钥匙扣 / 冰箱贴 / 徽章 / 麻将周边 |

**报告里发现同行用 IP 词时**：
- ✅ 标注 ⚠️ "侵权风险"
- ✅ 不能作为本店行动建议
- ✅ 解释"这家同行能上是因为授权 / 灰色，本店不能复制"

## R-006 · 验证码反爬（单次尝试规则）

**一句话**：撞验证码立即让 owner 手动过码，OCR 只允许调一次 API 试一次。

**详细解释**：
- 同 R-013 但角度不同：R-013 是工程层"不允许循环重试"
- R-006 是平台层"不允许任何形式的反爬"
- 1688 平台明确规定不允许第三方过码

**出错示例**：
- ❌ 接打码平台
- ❌ 训练专门的滑块识别模型
- ❌ 调用 GPT-Vision / Claude Vision 识别完直接 click

**替代方案**：
- ✅ 撞码 → 暂停 → 提示 owner 手动过 → 等"继续"指令
- ✅ 单次尝试 + 失败立即交人

## R-007 · 数据隔离（客户数据不进引擎仓库）

**一句话**：客户店铺的具体数据（询盘明细 / SKU 清单 / BSR 分等）只能写在客户项目目录，引擎仓库永远只放方法论。

**详细解释**：
- 引擎仓库 = `~/projects/_1688店铺体检引擎/`（共享给所有店铺）
- 店铺项目 = `~/projects/<店铺英文名>/`（每店独立）
- 引擎只能装"通用方法 / 模板 / 红线"
- 店铺数据（包括快照 / 报告 / 截图）只能进店铺项目

**出错示例**：
- ❌ 引擎 `examples/` 目录放艺漫坊询盘明细
- ❌ 引擎 README 引用 "金驰月销 1.5 万"
- ❌ 引擎 git commit 含具体店铺数据

**替代方案**：
- ✅ 引擎模板用 `{{店铺名}}` / `{{二级域名}}` 占位符
- ✅ 跨客户对比的方法论案例可以写"某客户"而不写真名（引擎层例外）
- ✅ 店铺数据全部进 `~/projects/<店铺>/02-数据快照/`

## R-008 · 资质合规（极限词 + 违规预警必现）

**一句话**：报告模块 1（店铺基础健康）必须扫极限词 + 列违规历史，不允许漏报。

**详细解释**：
- 1688 平台对"最 / 第一 / 唯一 / 国家级"等极限词扣分严
- 历史违规（活动违规 / 假货 / 描述不符）影响店铺加权
- 漏报 = owner 不知道风险 = 店铺继续踩坑

**出错示例**：
- ❌ 报告模块 1 只写资质和钻级，没扫极限词
- ❌ 违规记录页有数据但报告没写
- ❌ "本月违规 0 条"但实际有未处理违规

**替代方案**：
- ✅ 模块 1 必有 4 子项：资质 / 极限词扫描 / 违规历史 / 风控提醒
- ✅ 违规记录页（`work.1688.com/...ruleLllegal_new`）每次必抓
- ✅ 极限词扫商品标题 + 详情页 + 旺铺装修
