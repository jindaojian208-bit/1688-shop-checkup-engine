# 工程红线（Claude 工程层规则）

> 6 条工程红线，全部围绕「不能让自动化引擎变成失控的执行机器」。

## R-009 · 禁止存账号密码

**一句话**：账号密码（含手机号 / 验证码 / 短信码）任何形式都不能出现在引擎或店铺仓库里。

**详细解释**：
- 网页表单输入框：用户在 Chrome 里手动登录，引擎只读已登录的 cookie / session
- 环境变量：禁止 `.env` 含 1688_PASSWORD
- 明文配置：禁止 yaml / json / md 出现 "password: xxx"

**出错示例**：
- ❌ `~/projects/<店铺>/scripts/login.py` 含 `pwd = "xxx"`
- ❌ 控制台 README 写 "请把密码填到这里"
- ❌ git commit 包含含密码的文件

**替代方案**：
- ✅ owner 在 Chrome 里手动登录 1688 / 生意参谋 / P4P
- ✅ 引擎用 `mcp__plugin_superpowers-chrome` 复用已登录的浏览器
- ✅ owner 自己保管账号，引擎不碰

## R-010 · 禁止自动定时

**一句话**：launchd / cron / schedule / 任何后台周期运行 = 全禁。

**详细解释**：
- 月度体检是 owner 触发型，不是后台跑型
- 自动定时 = 失控 = 撞验证码 owner 不在场 = 账号风险

**出错示例**：
- ❌ `~/Library/LaunchAgents/com.1688checkup.plist`
- ❌ `crontab -e` 加 `0 9 1 * * /path/to/checkup.sh`
- ❌ Python `schedule` 库后台跑

**替代方案**：
- ✅ owner 说触发词 → Claude 跑
- ✅ engine 不绑日期，按需触发（B-7 拍板）

## R-011 · 禁止替用户点"投放"按钮

**一句话**：碰钱红线 — P4P 投放 / SKA 续约 / 任何花钱按钮，Claude 不能替 owner 点。

**详细解释**：
- 投放按钮 = 真扣钱
- 即使在 P4P 后台已登录，Claude 也只读不点
- owner 必须亲自点确认

**出错示例**：
- ❌ Claude 自动点 `p4p.1688.com` 的"立即投放"
- ❌ Claude 自动续 SKA 年框
- ❌ Claude 自动报名"超级工厂"扶持位

**替代方案**：
- ✅ Claude 列出建议 + 截图当前状态
- ✅ owner 自己在浏览器点确认
- ✅ 报告里写 "P0 立即处理：投放优化建议清单（owner 自己点确认）"

## R-012 · 禁止破坏性 git

**一句话**：`--force` / `--no-verify` / `reset --hard` 跨提交 = 全禁。

**详细解释**：
- `git push --force` 会覆盖远程历史
- `commit --no-verify` 跳过 hook（可能跳过密码扫描）
- `reset --hard <几个 commit 之前>` 会丢工作

**出错示例**：
- ❌ `git push --force origin main`
- ❌ `git commit --no-verify`
- ❌ `git reset --hard HEAD~5`

**替代方案**：
- ✅ 冲突时 `git pull --rebase` + 手动解决
- ✅ revert 错误 commit 而不是 reset
- ✅ tag 发版用 `git tag -a` 而不是 `--force`

## R-013 · 禁止用 claude code 反爬

**一句话**：撞验证码 = 立即让 owner 手动；不允许 OCR 循环 / 自动 click 绕过。

**详细解释**：
- 1688 的滑块 / 文字 / 图形验证码 = 平台反爬手段
- claude code 调 OCR 重试 = 反爬升级 = 账号被封风险
- 单次 API 尝试规则（`feedback_captcha_single_attempt`）

**出错示例**：
- ❌ 撞码后 `while True: try OCR -> click`
- ❌ 用 selenium 模拟人类滑动轨迹
- ❌ 用第三方打码平台批量过码

**替代方案**：
- ✅ 撞码 → 暂停脚本 → 提示 owner 手动过码 → owner 过完后说"继续"
- ✅ 单次 OCR 尝试失败立即交人

## R-014 · 数字结论必须贴来源

**一句话**：报告里所有数字（询盘转化率 24% / 月销 1.5 万 / BSR 4.1 等）必须贴页面链接或截图路径。

**详细解释**：
- 数字无来源 = owner 无法核对 = 报告变成"AI 编造"
- 来源必须是原始页面 URL / 截图 / 数据快照 MD 路径

**出错示例**：
- ❌ "询盘转化率 24%"（无来源）
- ❌ "BSR 4.1 分"（不知道哪个页面）

**替代方案**：
- ✅ "询盘转化率 24%（数据来源：`02-数据快照/2026-04-26/04_询盘客户.md`）"
- ✅ "BSR 4.1 分（截图：`02-数据快照/2026-04-26/screenshots/lighthouse-overview.png`）"
