# 1688 店铺体检引擎 · 红线总表

> 这份文件是 **任何 Claude 会话进入店铺月度体检项目前必读的合规索引**。
> 14 条红线统一编号 R-001 ~ R-014，分三类：方法论 / 平台合规 / 工程。
> 详细说明见各分类文件。

## 红线索引（一页扫完）

### 一、方法论红线（R-001 ~ R-004）→ 详见 `03_对比方法红线.md`

| 编号 | 一句话 | 触发场景 |
|---|---|---|
| **R-001** | 禁止跨客户对比（金驰 vs 艺漫坊不行 / 任意两店不行） | 月度报告里出现 "vs <其他客户店铺名>" 句式 |
| **R-002** | 同行抓「自然首页 3-5 个」，禁止三梯度（V3 已废弃） | 用 "实力商家梯度" / "超级工厂梯度" 当对比基准 |
| **R-003** | 同行对比必须 4 维度（优惠 / 营销 / 价格 / 服务承诺） | 4 维度任缺一项即为不合格 |
| **R-004** | 报告模块 6 必须呈现「自然首页同行 4 维度对比矩阵」 | 模块 6 没矩阵或矩阵 < 3 行 |

### 二、平台合规红线（R-005 ~ R-008）→ 详见 `02_平台合规红线.md`

| 编号 | 一句话 | 触发场景 |
|---|---|---|
| **R-005** | IP 授权风险：禁止建议本店上 IP 词 SKU；同行 IP 词必须标 ⚠️侵权风险 | 出现"建议上 鬼灭之刃 / TNT 少年团 / 时光代理人 / 谷子二次元 等 IP 词" |
| **R-006** | 验证码自动识别只调一次 API；失败立即让 owner 手动 | 撞码时循环重试 / 反爬绕过 |
| **R-007** | 禁止把客户店铺数据写进引擎仓库（数据隔离） | 引擎 git 提交里出现具体店铺数据 |
| **R-008** | 资质合规：极限词扫描 + 违规预警必须出现在报告模块 1 | 极限词 / 违规历史漏报 |

### 三、工程红线（R-009 ~ R-014）→ 详见 `01_工程红线.md`

| 编号 | 一句话 | 触发场景 |
|---|---|---|
| **R-009** | 禁止存账号密码（包括网页表单 / 环境变量 / 明文配置文件） | 引擎或店铺仓库出现 password/账号/手机号 |
| **R-010** | 禁止自动定时（launchd / cron / schedule 全禁） | 后台脚本周期跑月度体检 |
| **R-011** | 禁止替用户点"投放"按钮（碰钱红线） | claude code 自动点 P4P 投放确认 |
| **R-012** | 禁止破坏性 git（--force / --no-verify / reset --hard 历史） | git push --force / 跳过 hook |
| **R-013** | 禁止用 claude code 反爬（自动过验证码 / 自动点确认） | 撞码后 OCR 循环 / 自动 click |
| **R-014** | 报告里所有数字结论必须贴原始页面链接或截图路径 | 报告里"询盘转化率 24%"无来源 |
